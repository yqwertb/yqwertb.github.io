# 1 浏览器渲染的过程

解析HTML构建DOM树 => 解析CSS构建CSSOM树 => 将DOM树和CSSOM树合并为渲染树 => 布局渲染树 => 绘制渲染树

  • 布局渲染树：DOM树只规定了哪些元素可见和每个元素的计算规则，没有确定的位置和尺寸。所以从根节点开始遍历，确定每个元素的确切位置和尺寸。

# 2 重绘和回流

  • 重绘：只改变节点的外观而不改变布局的操作
  • 回流：节点的几何属性或者布局发生改变

引起回流的属性和方法

  • 添加或删除可见的DOM元素
  • 元素尺寸改变：margin、padding、border、width、height
  • 浏览器窗口尺寸改变
  • 计算offsetWidth offsetHeight属性
  • 设置style属性的值

# 3 数组去重

  • Array.indexOf() 检查元素在数组中第一次出现的位置是否和现在的位置相等（不处理NaN）
  • Array.includes() 处理NaN
  • Array.sort() 先进行数组元素的排序，然后遍历比较相邻元素
  • 使用ES6中的Set类型
  • Array.reduce(callback(pre, cur, index), src)执行自定义的函数 将元素加入src中 如果src中用array.indexOf()没有找到cur 就push进去
  • 使用Array.filter(callback(item, index, arr))和Object.hasOwnProperty() 如果(typeOf item + item)组成的字符串在对象中没存储，就加进去，用Object.hasOwnProperty()判断

# 4 作用域 执行环境

  • 变量作用域

    变量的作用域是指变量可以被使用的范围。

    在函数内部，变量的搜索是从内到外的，函数内部没有定义的变量，会沿着函数定义时的作用域往上搜索这个变量。

  • 变量对象 variable object

    每个执行环境下都有定义的变量，这种变量绑定了当前环境下的形参，变量，函数，理解为代码编译时产生的变量。

  • 活动对象 active object

    当函数执行的时候产生的变量，活动对象会替代变量对象

  • js执行上下文

    js的执行分为两个阶段：解释和执行

    • 解释
      1. 词法分析
      2. 语法分析
      3. 作用域规则的确定
    • 执行
      1. 创建执行上下文
      2. 执行函数代码
      3. 垃圾回收

    解释阶段会确定作用域链，this的指向就是执行上下文确定时绑定的

    区别：执行上下文在运行时确定，会改变，作用域在定义时确定，不会改变。

  • 作用域链

    自由变量：在当前作用域需要得到的变量，但却没有定义。此变量的值需要沿着作用域链向上寻找（函数创建的那个作用域）

# 5 什么是闭包

闭包是指一个函数可以访问另一个函数作用域中的变量

闭包是指一个函数在执行完后，它的词法作用域被保留下来，没有被销毁，作用域内的变量被其他的函数调用

# 深入问题

  1. 为什么其他非闭包的函数没有权限访问另一个函数的内部作用域

    因为函数在执行后作用域会被清理，内存也会被回收，所以访问不到

  2. 为什么闭包有这个权限

    因为在闭包函数的执行环境中，作用域链的顶层是闭包这个函数，紧接着就是闭包函数的上级作用域。当前作用域中寻找变量是沿着作用域链从当前作用域一直找到全局作用域，直到找到为止，所以上级作用域不会被清理。
