# CSS

# 1 BFC

# 1.1 BFC是什么意思，怎么生成，有什么用？

BFC是块格式化上下文，生成一块与外界隔离的区域

BFC生成方法有：float，绝对定位，display: inline-block | flex | table-cell，overflow: hidden 等

BFC的用途：阻止元素被浮动元素覆盖，包含浮动元素（解决子元素浮动造成父元素高度坍塌的问题），阻止相邻元素的margin合并

相关链接： https://segmentfault.com/a/1190000013023485

# 1.2 清除浮动的方法？

  • 在浮动元素后添加空div，并添加样式clear:both;height:0,;overflow:hidden;

  • 设置伪元素::after

  • 给父元素添加浮动或者设置高度

  • 给父元素添加overflow:hidden

# 3 伪类和伪元素

伪类通常用:表示，伪元素用::表示

最主要的区别是伪元素创建了文档树中不存在的元素，而伪类没有创建

# 4.2 rem、em的区别？

区别 rem em
计算规则 根据根元素的字体大小计算 根据离自己最近元素的字体大小计算
继承 只继承根元素的字体大小 如果元素本身设置字体大小，则其他属性继承本身的，如果本身没有设置就向上寻找设置字体大小的元素进行继承

# 5 Position的值？

# 5.1 Position都有哪些值？

值 描述
static 文档流的默认排版位置
relative 相对于本元素默认定位的定位
absolute 相对于本元素上层元素中，离本元素最近且具有定位的父辈元素的定位
fixed 相对于浏览器视口左上角的定位
sticky 类似relative和fixed的搭配使用，没达到设置的top等属性值时，元素在默认位置，达到设置的值后变为fixed

# 5.2 Position的top等属性中%是根据什么计算的？

relative: 根据最近的父级元素的width height计算

absolute: 根据最近的设置相对定位的父级元素的width height padding计算

fixed: 根据浏览器视口的width height计算

# 7 行内元素、块级元素和行内块元素的区别

# 行内元素

行内元素会按顺序在一行排列，排满一行后换行。行内元素不能设置width height，margin只能设置水平方向的值，竖直方向的不能生效。padding可以设置竖直方向的值 ，但是不会撑开元素的高度。 border可以正常设置。

# 块级元素

块级元素独占一行，可以设置width height margin padding

# 行内块元素

行内块元素不会自动换行，可以设置width height margin padding

# 8 如何设置小于12px的字体大小

  • zoom
    .span1{
        font-size: 12px;
        display: inline-block;
        zoom: 0.8;
    }
  • scale()
    .span1{
        font-size: 12px;
        display: inline-block;
        transform:scale(0.8);
    }

# 10 flex布局

在子元素定义的属性

flex-grow 元素放大

  • 只有一个元素设置

    值小于1，扩大的空间是剩余空间乘设置的比例

    值大于1，剩下的空间都被使用

  • 多个元素设置

    如果所有元素设置值的和小于1，各元素扩大的空间是剩余空间乘设置的比例

    如果所有元素设置值的和大于1，剩余空间都被利用，按照设置值分配

flex-shrink 元素缩放

  • 多个元素设置
